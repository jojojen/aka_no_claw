# /new 端到端鑑別度測試（E2E Discriminating Tests）

目的：用三題無法靠訓練資料矇混的問題，驗證 /new 從「蒐集資料」到「實作工具」
的整套流程。三題分別打中不同的管線段：

| | 打中的管線段 | 為什麼有鑑別度 |
|---|---|---|
| Q1 | API 抓數 → 時間對齊 → 多步計算 → 依據驗證 | 2026 年即時序列資料，計算需逐日掃描（running max），任何取值/窗口錯誤都會偏離 ground truth |
| Q2 | **/search 搜頁 → /fetch 抓頁 → 蒸餾 → 實作** | 2026 年的時事事實，訓練資料裡沒有正確值；不真的搜尋+抓頁就只能編造或拒答 |
| Q3 | **雙序列抓數 → 跨市場日期對齊 → 統計運算** | 台/美交易日曆不同（台股補班日、美國假日），不做共同日期交集對齊就會算錯或 crash；r 值無法背答案 |

## Q1 — `/new 算 0050 今年以來的最大回撤是多少%`

最大回撤（MDD）定義不在 codegen 種子裡（種子只有泛用開發技巧，遵守
no-formulas-in-knowledge-db 規則），公式概念模型可能知道，但**數值**
必須來自 Yahoo chart API 的 2026 年實際日線。

**通過標準**：
1. 答案的 MDD 與獨立計算的 ground truth（close 或 adjclose 任一口徑）誤差 ≤ 0.3 個百分點。
2. 計算依據印出實際資料點：峰值日期/價格、谷值日期/價格，且兩日期都落在 2026-01-01 之後、今天之前，峰先於谷。
3. 驗證 gate 全程把關（錯誤基期/造假依據要被 FAIL 進 repair loop，不可放行）。

**Ground truth 取得**：自寫獨立腳本抓
`https://query1.finance.yahoo.com/v8/finance/chart/0050.TW?period1=<2026-01-01>&period2=<now>&interval=1d`
逐日算 running max 與最大跌幅，close、adjclose 各算一份。

## Q2 — `/new 假設以日本銀行現在的政策金利年複利，100萬日圓存10年的本利和是多少日圓`

設計重點：**/search 單獨答不了**（要算複利）、**純 codegen 單獨也答不了**
（「現在的政策金利」是 2026 年的時事值，訓練資料沒有正確值）。
種子規則的參考頁（報酬率/YTD 等 Wikipedia 頁）與政策金利無關 → 規則接地落空
→ 觸發**搜尋接地 fallback**：LLM 把需求轉成一條搜尋查詢 →
`search_yahoo_japan_playwright`（與 /search 同一後端）→ 取前幾筆結果用
`fetch_page_text`（與 /fetch 同一抓取器）抓頁 → 蒸餾出政策金利現值、
生效時點與出處 → 注入 codegen prompt → 生成的程式用該利率算複利。

**通過標準**：
1. 所用利率與獨立查證（WebSearch 驗證的現行日銀政策金利）一致；
   不可用訓練資料裡的過期利率（如 -0.1% 時代值）。
2. 本利和 = 1,000,000 × (1+r)^10，與 ground truth 誤差 ≤ 1 日圓（捨入容許）。
3. 計算依據印出：所用利率、利率來源（URL/機構）與資訊時點、本金、年數。
4. 流程上確實走過搜尋接地（log 可見 search grounding 查詢與命中頁）。

## Q3 — `/new 算 0050 跟 QQQ 今年以來日報酬的相關係數`

設計重點：使用者點名的型態——**找到資料後，由生成的程式碼做實質運算才得到答案**。
兩條序列來自不同市場（0050.TW 台股、QQQ 美股），交易日曆不一致：

- 台灣有補班交易日、美國有 MLK/總統日等假日 → 兩序列**不等長、日期不對齊**。
- 正確做法：以日期為 key 取交集、排序後再算日報酬與 Pearson r。
- 偷懶做法（直接 zip 兩列價格）會得到明顯偏離的 r 或長度錯誤——這就是鑑別點。

**通過標準**：
1. 答案明示口徑（日報酬相關），r 與 ground truth 誤差 ≤ 0.05。
   若模型自行改算價格水準相關（r≈0.90），視為口徑跑題 → FAIL。
2. 計算依據印出：對齊後的共同交易日數（≈100±5）、期間（2026-01-02 → 最近交易日）、
   兩個資料來源（Yahoo chart API 的兩個 symbol）。
3. 不可硬編 r 值；數據必須來自 API 實抓。

**Ground truth 取得**：獨立腳本 /tmp/gt_corr.py——抓兩 symbol 的 2026 YTD 日線，
date→price 映射取交集排序，算價格水準與日報酬兩種 Pearson r（close/adjclose 各一）。

## 搜尋預算鐵則（②不被封鎖）

- /new 的搜尋接地**每請求最多 1 條查詢**，**每日上限 4 條**（持久化計數，
  存 `tools_dir` 旁的 `new_search_state.json`），超限直接跳過接地（fail-open，
  /new 仍可生成，只是沒有搜尋知識）。
- 蒸餾結果以查詢字串為 key 快取 24h：重跑同題（含測試迭代）不再耗預算、
  不再打 Yahoo。
- 此上限獨立於 /search 指令本身的人工使用；兩者合計仍須遵守「每日個位數」
  的全局原則。

## Ground truth（2026-06-11 確立）

- Q1：MDD = **−10.84%**，峰 2026-02-26（81.15）→ 谷 2026-03-31（72.35），
  close 與 adjclose 同值（獨立腳本 /tmp/gt_mdd.py 計算）。
- Q2：日銀政策金利現值 = **0.75%**（2026-04 會合維持；6/16 會合僅「檢討」升至
  1.0%，尚未生效——答 1.0% 即為被預期新聞誤導，FAIL）。
  本利和 = 1,000,000 × 1.0075^10 ≈ **1,077,583 円**。
- Q3：對齊共同交易日 **100 天**（2026-01-02 → 2026-06-09），
  日報酬 r = **0.1697**（close）/ **0.1715**（adjclose）；
  價格水準 r = 0.9062 / 0.9014（口徑跑題參考值，非通過值）。

## 執行方式

每題用 /tmp 腳本直跑 `DynamicToolRunner.run_detailed`（隔離 tools_dir，
不汙染正式 manifest），結果與 log 寫 /tmp/live_new_q12.out；
未達標就修管線（泛用修法，不寫死領域知識）再跑，直到**三題同輪都過**。
